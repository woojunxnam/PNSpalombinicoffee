# Contributing Guide

## 기본 원칙
- 이 저장소는 정적 웹페이지 유지보수 중심입니다.
- 기능 추가보다 **링크 안정성, 문서 품질, 배포 안전성**을 우선합니다.
- 불필요한 대규모 리팩터링은 지양합니다.

## 작업 전 확인
1. `index.html` 기준으로 전체 페이지 구조를 파악합니다.
2. 링크/에셋 상대경로가 깨지지 않는지 확인합니다.
3. 변경 범위를 최소화합니다.

## 변경 후 확인
다음 명령으로 로컬 확인을 권장합니다.

```bash
python -m http.server 8000
```

그리고 주요 페이지를 수동 점검합니다.
- `/`
- `/products/`
- `/machines/`
- `/contact.html`

## 커밋/기록
- 변경 목적이 분명한 작은 단위 커밋을 권장합니다.
- 사용자에게 보이는 변경 또는 유지보수성 변경은 `CHANGELOG.md`에 기록합니다.
