# CSS 무결성 가드

이번 사이트는 전역 CSS 파일 하나가 여러 페이지 레이아웃을 함께 제어합니다.  
그래서 작은 문법 오류라도 제품 목록, 머신 목록, 상세페이지 전체에 연쇄적으로 영향을 줄 수 있습니다.

## 왜 이 점검이 필요한가
- `content:` 문자열 따옴표가 닫히지 않으면 이후 CSS가 연쇄적으로 깨질 수 있습니다.
- 세미콜론 또는 중괄호 누락은 전역 레이아웃 규칙 전체를 무시하게 만들 수 있습니다.
- 문자 치환 또는 인코딩 문제는 pseudo-element 아이콘과 주석뿐 아니라 파서 안정성에도 영향을 줄 수 있습니다.

## 실행 방법
```bash
python scripts/check_css.py
```

정상일 때:
- `CSS 무결성 검사 통과`

실패할 때:
- 어떤 CSS 파일의 몇 번째 줄이 문제인지 출력합니다.

## 이 스크립트가 잡는 것
- UTF-8 인코딩으로 읽을 수 없는 CSS 파일
- 닫히지 않은 문자열
- 닫히지 않은 주석
- 닫히지 않은 중괄호 / 과도한 닫는 중괄호
- `content:` 선언의 따옴표/세미콜론 오류

## CSS 수정 시 주의사항
- `content:` 값은 반드시 문자열 또는 표준 키워드로 끝까지 닫아야 합니다.
- pseudo-element(`::before`, `::after`) 수정 시 따옴표와 세미콜론을 먼저 확인합니다.
- 큰 전역 파일을 수정할 때는 주석/중괄호 짝이 맞는지 같이 봅니다.
- 인코딩이 흔들릴 수 있는 편집기/복사 과정에서는 특수문자보다 ASCII 또는 Unicode escape를 우선 검토합니다.

## 최소 수동 확인 페이지
- 홈페이지: `index.html`
- 제품 목록: `products/index.html`
- 머신 목록: `machines/index.html`
- 볼케이노 루비 상세: `products/volcano-ruby.html`

## 최소 수동 확인 포인트
- 데스크탑:
  - 헤더/히어로 폭이 정상인지
  - 카드 그리드가 2~3열로 정상 정렬되는지
  - 상세페이지 좌우 레이아웃이 무너지지 않는지
- 모바일:
  - 햄버거 메뉴가 정상 열리는지
  - sticky CTA와 scroll-top이 겹치지 않는지
  - 제품/머신 목록 카드가 1열로 자연스럽게 쌓이는지
